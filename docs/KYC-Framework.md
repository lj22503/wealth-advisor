# KYC（了解你的客户）框架 - 理财经理投顾系统

## 概述

KYC（Know Your Customer）是财富管理中的核心环节，帮助理财经理全面了解客户，提供个性化的投资建议。本框架为理财经理投顾系统设计，包含完整的客户了解流程和工具。

## 1. KYC核心维度

### 1.1 基本信息
- **个人资料**：年龄、职业、收入、家庭状况
- **财务状况**：资产规模、收入来源、负债情况
- **投资经验**：投资年限、产品熟悉度、风险认知

### 1.2 风险偏好评估
#### 风险承受能力（客观）
- **资产规模**：可投资资产总额
- **收入稳定性**：收入来源和稳定性
- **负债情况**：房贷、车贷等负债比例
- **家庭责任**：抚养子女、赡养老人等责任

#### 风险容忍度（主观）
- **投资目标**：保值、增值、高收益
- **亏损接受度**：最大可接受亏损比例
- **投资期限**：短期、中期、长期
- **流动性需求**：资金使用计划

### 1.3 投资目标分析
| 目标类型 | 时间框架 | 预期收益 | 风险等级 |
|---------|----------|----------|----------|
| 资产保值 | 1-3年 | 3-5% | 低风险 |
| 稳健增值 | 3-5年 | 5-8% | 中低风险 |
| 财富增长 | 5年以上 | 8-12% | 中高风险 |
| 高收益投资 | 5年以上 | 12%+ | 高风险 |

### 1.4 行为特征分析
- **投资风格**：保守型、稳健型、平衡型、成长型、进取型
- **决策模式**：自主决策、依赖建议、跟随市场
- **关注重点**：收益、风险、流动性、税收优化
- **沟通偏好**：频率、方式、内容深度

## 2. KYC数据收集工具

### 2.1 客户信息表
```json
{
  "personal_info": {
    "name": "张三",
    "age": 35,
    "occupation": "互联网产品经理",
    "annual_income": 500000,
    "family_status": "已婚有子女"
  },
  "financial_status": {
    "total_assets": 3000000,
    "investable_assets": 1500000,
    "monthly_expense": 20000,
    "liabilities": 500000
  },
  "investment_experience": {
    "years": 5,
    "products": ["股票", "基金", "银行理财"],
    "risk_awareness": "中等"
  }
}
```

### 2.2 风险评估问卷
**问卷维度**：
1. 投资目标（权重30%）
2. 风险承受能力（权重40%）
3. 投资经验（权重20%）
4. 流动性需求（权重10%）

**评分标准**：
- 0-30分：保守型投资者
- 31-60分：稳健型投资者
- 61-80分：平衡型投资者
- 81-100分：成长型投资者

### 2.3 投资偏好调查
- **产品偏好**：股票、债券、基金、保险、另类投资
- **市场偏好**：A股、港股、美股、其他市场
- **行业偏好**：科技、消费、医药、金融、制造
- **风格偏好**：价值投资、成长投资、指数投资

## 3. KYC分析模型

### 3.1 客户画像生成
基于收集的数据，系统自动生成客户画像：

**示例画像**：
```
客户类型：成长型投资者
风险等级：中高风险
投资目标：财富增长（5年+，预期收益8-12%）
资产配置建议：股票60%，基金30%，债券10%
关注行业：科技、消费、新能源
```

### 3.2 风险匹配度分析
将客户风险偏好与产品风险等级进行匹配：

| 客户风险等级 | 适合产品风险等级 | 匹配度 |
|-------------|-----------------|--------|
| 保守型 | R1-R2 | 高 |
| 稳健型 | R2-R3 | 高 |
| 平衡型 | R3-R4 | 高 |
| 成长型 | R4-R5 | 高 |

### 3.3 投资建议生成
基于KYC分析，生成个性化投资建议：

1. **资产配置方案**：股债比例、行业分布、地域分布
2. **产品推荐**：具体基金、股票、理财产品
3. **投资策略**：定投策略、择时策略、再平衡策略
4. **风险控制**：止损点、仓位控制、分散投资

## 4. KYC在投顾系统中的应用

### 4.1 客户导入流程
```
Excel导入 → 基本信息提取 → 风险评估 → 画像生成 → 建议生成
```

### 4.2 数据更新机制
- **定期更新**：每半年更新一次KYC信息
- **事件触发**：重大生活变化时更新
- **自动提醒**：系统提醒理财经理跟进

### 4.3 报告生成
基于KYC数据生成客户报告：
- **客户画像报告**：全面展示客户特征
- **风险评估报告**：详细的风险评估结果
- **投资建议报告**：个性化的投资方案
- **定期回顾报告**：投资表现与KYC匹配度分析

## 5. 技术实现

### 5.1 数据模型
```typescript
interface KYCData {
  // 基本信息
  personalInfo: PersonalInfo;
  financialStatus: FinancialStatus;
  investmentExperience: InvestmentExperience;
  
  // 风险评估
  riskAssessment: RiskAssessment;
  investmentGoals: InvestmentGoal[];
  
  // 行为特征
  behaviorProfile: BehaviorProfile;
  
  // 分析结果
  customerProfile: CustomerProfile;
  investmentRecommendations: Recommendation[];
}
```

### 5.2 服务接口
```typescript
// KYC服务接口
interface KYCService {
  // 数据收集
  collectKYCData(clientId: string, data: KYCData): Promise<void>;
  
  // 风险评估
  assessRisk(clientId: string): Promise<RiskAssessmentResult>;
  
  // 画像生成
  generateProfile(clientId: string): Promise<CustomerProfile>;
  
  // 建议生成
  generateRecommendations(clientId: string): Promise<Recommendation[]>;
  
  // 报告生成
  generateReport(clientId: string, reportType: string): Promise<Report>;
}
```

### 5.3 前端组件
- **KYC信息收集表单**：多步骤表单收集客户信息
- **风险评估问卷**：交互式问卷评估风险偏好
- **客户画像展示**：可视化展示客户特征
- **投资建议展示**：个性化建议展示和调整

## 6. 合规要求

### 6.1 监管要求
- **适当性管理**：确保产品与客户风险承受能力匹配
- **信息披露**：充分披露产品信息和风险
- **记录保存**：保存KYC记录至少5年
- **定期评估**：定期重新评估客户风险承受能力

### 6.2 数据安全
- **数据加密**：敏感信息加密存储
- **访问控制**：严格的权限管理
- **审计日志**：完整的操作记录
- **数据脱敏**：展示时脱敏处理

## 7. 最佳实践

### 7.1 数据收集技巧
- **渐进式收集**：分多次收集，避免一次性过多问题
- **场景化提问**：结合具体场景提问，提高准确性
- **交叉验证**：通过多个问题验证同一维度
- **动态调整**：根据已有信息动态调整后续问题

### 7.2 沟通策略
- **建立信任**：通过专业和透明建立信任
- **教育客户**：帮助客户理解风险和收益
- **定期沟通**：定期回顾和更新KYC信息
- **个性化服务**：基于KYC提供个性化服务

### 7.3 持续优化
- **反馈收集**：收集客户对KYC流程的反馈
- **模型优化**：基于实际数据优化评估模型
- **流程改进**：持续改进KYC收集和分析流程
- **技术升级**：利用新技术提升KYC效率
